## 1. 设计目标

Maestro 是基于 Lua 的多智能体编排运行时。Web UI 的核心价值：

- **编排过程可视化**：Phase → Agent 层级结构、实时进度、token 消耗一目了然
- **运行历史管理**：列表、筛选、检索历史 Run，查看报告与 Findings
- **工作流管理**：浏览、编辑、保存 Lua 编排脚本
- **低使用门槛**：本地启动即用，无需额外部署

设计原则：

| 原则 | 说明 |
|------|------|
| 信息密度优先 | 开发者工具，屏幕空间利用率 > 留白美感 |
| 实时反馈 | 运行中状态通过动效传达，无需手动刷新 |
| 层级清晰 | 列表 → 详情的钻取路径短且可逆 |
| 克制装饰 | 数据和状态是主角，视觉元素服务于信息传达 |

## 2. 视觉风格

### 2.1 风格方向：可观测平台（Observability Dashboard）

参考产品：Grafana、Datadog、Honeycomb、Linear。

选择理由：
- Maestro 核心场景是**监控多 Agent 编排过程**（Phase/Agent 层级、token 统计、收敛验证），属于数据可视化场景
- 需要承载高信息密度（事件流、Agent 卡片网格、进度条）
- 开发者受众，偏好专业、克制的视觉语言

### 2.2 配色方案

暗色优先（Dark-first），契合开发者工具定位。

#### 背景层级

| Token | 色值 | 用途 |
|-------|------|------|
| `bg-base` | `#0a0e14` | 页面最底层背景 |
| `bg-surface` | `#141821` | 卡片、面板、表格行背景 |
| `bg-elevated` | `#1c2230` | 悬浮元素、抽屉、模态框 |
| `bg-hover` | `#242b3d` | hover 状态叠加 |

#### 前景与文本

| Token | 色值 | 用途 |
|-------|------|------|
| `text-primary` | `#e4e7eb` | 主要文本（标题、正文） |
| `text-secondary` | `#8b95a7` | 次要文本（时间戳、标签） |

### 2.3 字体方案

| 用途 | 字体 | 回退栈 |
|------|------|--------|
| 标题/UI | **Space Grotesk** | `-apple-system, system-ui, sans-serif` |
| 数据/代码 | **IBM Plex Mono** | `JetBrains Mono, Menlo, monospace` |
| 正文 | **Inter** | `system-ui, sans-serif` |

> Space Grotesk 提供 slightly geometric 的科技感标题，区别于默认 Inter/Roboto。
> IBM Plex Mono 用于 token 数字、agent ID、Lua 代码、时间戳，保证数据等宽对齐。

| `text-lg` | 16px | 卡片标题、区块标题 |
| `text-xl` | 20px | 页面标题 |
| `text-2xl` | 28px | 数据大数字 |

## 3. 页面结构与导航

### 3.1 全局导航

四个顶层页面，TopNav 常驻：

```
┌─────────────────────────────────────────────────────────────┐
│  ◉ maestro    Dashboard   Runs   Workflows   Backends   ▶ Run │
└─────────────────────────────────────────────────────────────┘
```

| 页面 | 路由 | 职责 |
|------|------|------|
| Dashboard | `/` | 全局总览，活跃 Run + 最近完成 + 今日统计 |
| Runs | `/runs` | Run 列表，筛选/搜索，点击进详情 |
| Workflows | `/workflows` | Lua 脚本库，编辑/保存/试运行 |
| Backends | `/backends` | LLM provider 配置管理 |

### 3.2 层级流转

```
Dashboard ──────┐
                ├──▶ Runs 列表 ──▶ Run 详情 (/runs/:id)
Workflows ──────┘                  │
                                  │
                          ┌───────┴───────┐
                          │               │
                     Events 视图     Findings 视图
                          │               │
                          └───────┬───────┘
                                  │
                            Report 视图
```

- Run 详情通过面包屑 `Runs › #r4f2a` 保持层级感
- `‹ 返回列表` 按钮始终可见
- 全局 `▶ Run` 按钮在任何页面均可发起新任务

### 3.3 全局元素

| 元素 | 位置 | 行为 |
|------|------|------|
| TopNav | 固定顶部 | 当前页高亮（accent 下划线） |
| `▶ Run` 按钮 | TopNav 右端 | 点击弹出 Run 对话框（选 Workflow + 输入 task） |
| 面包屑 | 详情页顶部 | 显示路径层级 |
| 通知 Toast | 右下角 | Run 完成/失败通知（非阻塞，5s 自动消失） |

**列定义：**

- 运行中的行：status 徽章脉冲，phase 进度条实时更新
- 表格支持滚动（固定表头）

| 区块 | 内容 |

| TIMELINE | Phase 列表，带状态点（✓●○），点击滚动到对应 Phase |
| PHASE INFO | 当前选中 Phase 的元信息（角色类型、agent 数量） |
| FINDINGS | 已产出 Findings 数量，点击切换到 Findings 视图 |
| REPORT | 报告状态（not ready / 已生成），点击查看 |

#### 主区域上半：Phase 展开区

- 每个 Phase 是手风琴（Accordion）节点，默认展开当前运行中的 Phase
- Phase 内是 **Agent 卡片网格**（自动换行，每张卡 ~200×120px）
- Agent 卡片内容：角色标签（Producer/Adversary/Voter）、状态、token、耗时
- 收敛验证的 Agent 卡片按角色分组（用分隔线 + 角色标签标注）

#### 主区域下半：事件流

- 实时滚动的事件日志，等宽字体
- 缩进表示层级（Run → Phase → Agent）
- `▶` = 开始事件，`←` = 完成事件，`→` = 进度事件
- 支持 **暂停滚动**（底部 pause 按钮）
- 支持滚动到顶部查看历史

## 5. 交互细节与状态系统

### 5.1 状态徽章规范

| 状态 | 图标 | 颜色 | 文字 | 动画 |
|------|------|------|------|------|
| Running | ● | `#3b82f6` | 运行中 | 图标脉冲（1.5s 呼吸） |
| Success | ✓ | `#00e676` | 完成 | 无 |
| Failed | ✗ | `#ef4444` | 失败 | 出现时短暂闪烁红光 |
| Pending | ○ | `#6b7280` | 等待中 | 无 |
| Cancelled | ⊘ | `#f59e0b` | 已取消 | 无 |

### 5.2 进度条

```
  Phase 2/3
  ████████░░░░░░░░░░░░  40%
  └─────────────────┘  └── 百分比（等宽字体，右对齐）
  已完成（accent 色）    未完成（bg-elevated 色）
```

- 运行中：进度条末端有渐变光流动效（shimmer）
- 完成时：从当前位置平滑填充到 100%（300ms ease-out）

### 5.3 Agent 卡片状态

| 状态 | 视觉 |
|------|------|
| 等待中 | 灰色边框，半透明内容 |
| 运行中 | 蓝色左边框（3px），状态图标脉冲，token 数字平滑滚动 |
| 完成 | 绿色状态图标，正常显示 |
| 失败 | 红色边框，底部显示错误摘要（1 行） |

### 5.4 Agent 卡片密度切换

Run 详情页右上角提供密度切换：

| 模式 | 卡片尺寸 | 内容 |
|------|----------|------|
| Compact（默认） | 200×100px | 角色标签、状态图标、token、耗时 |
| Comfortable | 240×160px | 上述内容 + prompt 前 2 行 + output 摘要 1 行 |

### 5.5 事件流

- 新事件自动追加到底部，视图自动滚动跟随（sticky bottom）
- `⏸ pause`：暂停自动滚动，新事件标记未读数量
- 事件行缩进规则：Run 级无缩进，Phase 级缩进 2 空格，Agent 级缩进 4 空格
- 时间戳格式：`HH:MM:SS`（当天）或 `MM-DD HH:MM`（跨天）

### 5.6 实时更新策略

| 数据 | 更新方式 | 频率 |
|------|----------|------|
| Run 状态/进度 | WebSocket 推送 | 事件驱动（实时） |
| Agent token 计数 | WebSocket 推送 | 每 ~500ms 增量 |
| Runs 列表 | WebSocket 推送 + 页面可见时刷新 | 事件驱动 |
| Dashboard 活跃卡片 | WebSocket 推送 | 事件驱动 |

## 8. 响应式策略

### 断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Desktop | ≥1200px | 完整布局（本文档所有线框图） |
| Tablet | 768–1199px | 左侧栏收窄为图标条，Agent 网格 2 列 |
| Mobile | <768px | 单列堆叠，Agent 卡片全宽，左侧栏折叠为抽屉 |

### 优先级

Maestro Web UI 以 **Desktop 优先**（开发者工具，通常在大屏使用）。
Mobile 仅保证可读（查看 Run 状态/进度），不保证可编辑（Workflow 编辑器不适配小屏）。
